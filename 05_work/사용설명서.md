# 시스템 로그 분석 시스템 사용설명서

## 1. 개요
- 본 웹 애플리케이션은 서버·보안 장비에서 수집한 텍스트 로그를 업로드하면 자동으로 통계, 시각화, 보안 이슈, 에러 패턴, IP 별 행위를 분석해 주는 단일 페이지 도구입니다.
- 브라우저에서만 동작하며, 로컬에서 선택한 파일의 내용을 브라우저 메모리 내에서만 처리하므로 별도 서버 전송은 없습니다.

## 2. 시스템 요구 사항
- 최신 버전의 Chromium 기반 브라우저(Chrome, Edge) 또는 Firefox, Safari.
- HTML/CSS/JavaScript만 사용하므로 추가 프로그램 설치 불필요.
- 분석 대상 로그 파일 확장자: `.log`, `.txt`.

## 3. 로그 파일 준비
- 한 줄이 아래 형식을 만족해야 정상 파싱됩니다.

```text
YYYY-MM-DD HH:MM:SS LEVEL IP ACTION 상세내용
예) 2024-02-12 10:23:45 WARNING 10.0.0.5 LOGIN user=kim status=FAILED
```

- LEVEL: `INFO`, `WARNING`, `ERROR`, `CRITICAL` 중 하나.
- IP: IPv4 형식, ACTION: LOGIN/LOGOUT/ACCESS 등 한 단어.
- 상세내용에는 `status=FAILED`, `status=BLOCKED`, `status=403` 등 키워드가 포함될 경우 자동으로 보안/패턴 분석에 활용됩니다.

## 4. 사용 방법
1. `로그 파일 선택` 버튼을 눌러 분석할 파일을 지정합니다. 올바른 형식이면 버튼 텍스트에 자동으로 로그 개수가 표시되고 `분석 시작` 버튼이 활성화됩니다.
2. `분석 시작`을 클릭하면 결과 영역이 열리면서 다음 분석이 순차 실행됩니다.
3. 결과 영역 각 섹션을 스크롤로 내려 확인합니다. 필요 시 `상세 로그` 영역의 `모든 레벨` 드롭다운을 사용해 특정 레벨만 필터링할 수 있습니다.

## 5. 분석 결과 해석
- **📈 요약 통계**: 총 로그 수와 레벨별 건수, 급증 여부를 즉시 확인합니다.
- **📊 로그 레벨 차트**: 레벨별 분포를 막대 차트로 시각화합니다.
- **⏰ 시간대별 분석**: 0~23시 기준 로그 발생량 라인 차트와 시간대별 요약 카드가 제공돼 피크 시간을 파악할 수 있습니다.
- **🔒 보안 이슈**: 다수의 로그인 실패, 차단 계정, 403 응답, 외부 IP 접근, CRITICAL 이벤트 등을 자동 탐지해 심각도별 카드로 표시합니다.
- **🔍 에러 패턴 분석**: `로그인 실패`, `계정 차단`, `접근 권한 없음`, `인증 실패`, `TIMEOUT`, `Database` 등 패턴을 그룹화해 건수와 예시 로그를 제공합니다.
- **🌐 IP 주소 분석**: IP별 총 로그 수와 레벨별 건수를 보여주며 위험 징후(IP에 WARNING/ERROR가 많을 때)는 붉은 배경으로 표시됩니다.
- **📝 상세 로그**: 필터링된 로그를 원문 그대로 확인할 수 있으며, 타임스탬프/레벨/액션/상세내용이 한 줄씩 정리됩니다.

## 6. 문제 해결 가이드
- **분석 시작 버튼이 비활성화된 경우**: 로그 형식이 맞는지 확인하세요. 공백 라인을 제거하고 위 3장의 포맷을 다시 맞춰 저장합니다.
- **보안 이슈가 모두 “탐지되지 않음”으로 나오는 경우**: 로그에 `status=` 키워드나 `CRITICAL` 레벨이 포함되어 있는지 확인하세요.
- **차트가 비어 있는 경우**: 로그 개수가 매우 적거나 레벨/시간 정보가 누락되어 있을 수 있습니다. 포맷을 다시 검토하세요.

## 7. 유지보수 팁
- `system_access.log` 같은 최신 로그 파일을 주기적으로 내려받아 동일한 형식으로 저장하면 이전 분석과 결과를 비교하기 쉽습니다.
- 필요 시 `index.html`, `style.css`, `script.js`를 수정해 자체 로고/컬러/분석 규칙을 확장할 수 있습니다. JavaScript의 `detectSecurityIssues` 함수에 새로운 규칙을 추가하면 사용자 정의 경보를 쉽게 늘릴 수 있습니다.

문의 사항이나 커스터마이징 요청은 시스템 관리자에게 전달하십시오.
